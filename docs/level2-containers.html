Containers

Zoom into the system box. A “container” is a separately deployable/runnable thing or, here, a separately-buildable Swift unit. Thunderbird for iOS is one process composed of three units: the app target, the Core engine package, and the Bolt design system.

The three containers

The only runnable target. A pure-SwiftUI app entered at App.swift (@main), which holds the root state — an Accounts model and a FeatureFlags instance — and injects them through the SwiftUI environment. ContentView branches on whether any authorized account exists: if not, it shows onboarding; if so, the mailbox UI.

Responsibilities

  • Onboarding & account setup flow (WelcomeScreen → ManualAccount navigation stack)
  • Mailbox navigation (DrawerView) and the email list / reading screens
  • OAuth2 and password credential capture
  • Feature-flag gating & debug tooling
  • Localization (69 .lproj bundles) and app assets

It depends on both packages: import Account / Autoconfiguration / EmailAddress from Core, and BoltUI for styling. It contains no networking code of its own.

Detail See Level 3 · App components.

A UI-free Swift package that contains everything about talking to mail servers. It exposes nine products, layered so that protocol modules are independent and the Account module composes them into a unified API. The umbrella Core product simply @_exported import Account.

Transport split: IMAP & SMTP use SwiftNIO (streaming, long-lived connections, IDLE); JMAP & Autoconfiguration use URLSession (request/response HTTP). All public operations are async.

Detail See Level 3 · Core components.

A small SwiftUI-only package providing the visual language. Three products in a tidy chain: Editor (rich-text editing primitives) → BoltUI (re-exports Editor, adds the design tokens) → Bolt (umbrella).

What's inside BoltUI

  • Color.swift — a semantic palette: neutral, intent colors (primary, secondary, success, warning, critical) each with soft/default/hover/pressed steps, text roles, and an accent set — all with light & dark variants.
  • Font.swift — a Font.Style scale (titleLarge…caption) carrying size, line spacing, kerning, and weight, plus a .textStyle(_:) view modifier that honors Dynamic Type.
  • Editor — placeholder module reserved for the rich-text compose editor.

No external dependencies and no Core dependency — Bolt is purely presentational.

Dependency direction

Dependencies point strictly downward — the app knows about the packages; the packages never know about the app, and Bolt never knows about Core.

Protocol & format modules
IMAP · SMTP · JMAP → depend on MIME + EmailAddress · MIME & EmailAddress are leaf modules
✅ Acyclic by construction Because Core is UI-free and Bolt is Core-free, both packages build, test, and (in principle) ship independently of the app. This is what keeps the 13k-line Core/Tests suite runnable without a simulator UI.
