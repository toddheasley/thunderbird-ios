CI pipelines

Thirteen GitHub Actions workflows under .github/workflows/. The core gates run on every push/PR to main.

test.yml

Xcode Build. On push/PR to main, macos-26: resolve deps, build, analyze, boot an iPhone 17 simulator, run tests, archive (unsigned Release).

swift-format.yml

Swift Format. swift-format lint . --parallel --recursive --strict — formatting is enforced, not advisory.

shippable-builds.yml

TestFlight. Weekly + manual; Xcode 26.0 on macos-26, thunderbird_release environment, calls the fastlane beta lane.

deploy-docs.yml

Docs. Builds & publishes the developer documentation.

Plus PR automation: pulls-auto-assign, pulls-merged, and the needinfo-* trio.

Release (fastlane)

Two lanes in fastlane/Fastfile:

beta

Build & upload to TestFlight. Build number is a timestamp (YYYYMMDDHHmm); handles signing & provisioning.

release

Build & submit to the App Store.

Localization

69 language bundles under Thunderbird/Thunderbird/Localization/*.lproj — from Arabic and Chinese (Simplified & Traditional) to Kabyle, Esperanto, and Welsh — managed through Thunderbird's translation tooling. The UI is built string-key-first so new locales drop in without code changes.

Documentation

  • These C4 docs — static HTML in docs/ (Tailwind + shadcn-styled components).
  • Developer handbook — an mdBook (docs/book.toml): “Thunderbird for iOS — Developer Documentation,” with mermaid, page-ToC, and last-changed preprocessors, deployed via deploy-docs.yml.
  • In-tree DocC — Thunderbird/Thunderbird/Documentation.docc/: Thunderbird.md, Design.md, Database.md, Localization.md.
