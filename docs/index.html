Thunderbird for iOS

A native SwiftUI email client built on a protocol-complete email engine (IMAP, SMTP, JMAP, MIME) and a custom design system. This documentation follows the C4 model — four zoom levels from a 50,000-foot system context down to the inner workings of individual types.

🧭 How to read these docs Start at Level 1 (Context) for the big picture, then descend one level at a time. Every section below is a collapsed accordion — click to expand, or use Expand all. The left sidebar tracks your position.

Introduction

Thunderbird for iOS is organized as a workspace of three independently-buildable units:

Core

The email engine. Pure Swift, no UI. Implements IMAP, SMTP, JMAP, MIME, autoconfiguration, and account/credential management on top of SwiftNIO.

Bolt

The design system. Semantic colors, typography scale, and a rich-text Editor module shared across the UI.

Thunderbird

The app. The SwiftUI iOS target — onboarding, account setup, mailbox navigation, and email reading — consuming Core and Bolt.

The C4 levels

Each level zooms in one step. Higher levels answer “what and why”; lower levels answer “how.”

Level 1 · 50,000 ft
System Context
The user, the app, and the external mail systems it talks to — IMAP/SMTP servers, JMAP servers, OAuth2 identity providers, and autoconfig sources.
Level 2 · 10,000 ft
Containers
The three build units — the SwiftUI app, the Core engine package, and the Bolt design system — plus the technology in each.
Level 3 · 1,000 ft
Components
Inside each container: the Core modules (IMAP/SMTP/JMAP/MIME/Account/Autoconfig) and the app's feature areas and view models.
Level 4 · ground level
Code
The inner workings — IMAPClient command execution, the SMTP send state machine, JMAP batch requests, MIME parsing, and account/keychain bridging.
Cross-cutting
Runtime Flows
End-to-end sequences: adding an account, autoconfiguration, OAuth2 sign-in, fetching a mailbox, and sending a message.
Operations
Build & CI
Workspace layout, schemes, the test plan, the GitHub Actions pipelines, security scanning, and release via fastlane.
