Code — inner workings

Ground level. The actual types, their methods, and the mechanics that make the engine work — command execution over NIO, the SMTP state machine, JMAP batching, MIME decoding, and how credentials cross into the keychain. Signatures here are taken from the source.

📍 This is the deep end The C4 model treats Level 4 as optional and volatile — code changes faster than architecture. Treat the snippets as a faithful map of intent; the source is ground truth. File paths are given for every type.

IMAPClient is a reference type wrapping a NIO Channel and an EventLoopGroup. State is minimal: the Server config, a discovered Set<Capability>, and an optional idleHandler.

public class IMAPClient {
    public let server: Server
    public private(set) var capabilities: Set<Capability> = []
    public var isConnected: Bool { channel != nil && channel!.isActive }
    public var isIdling: Bool   { idleHandler != nil }

    public func connect() async throws { … }                 // bootstrap + refreshCapabilities()
    public func login() async throws                         // uses Server credentials
    public func login(username:password:) async throws
    public func list(wildcard:) async throws -> [(Mailbox, Mailbox.Status?)]
    public func select(mailbox:) async throws -> Mailbox.Status
    public func fetch(_ set: SequenceSet = .all,
                      attributes: [FetchAttribute] = .header) async throws -> MessageSet
    public func fetch(uid: UID, attributes: [FetchAttribute] = .complete) async throws -> Message
    @discardableResult public func idle() async throws -> AsyncStream<IdleEvent>
}

The command runner

Every operation funnels through one generic method:

private func execute<T: IMAPCommand>(command: T) async throws -> T.Result

Each command type (LoginCommand, ListCommand, FetchCommand, …) conforms to IMAPCommand, declaring its NIO handler and its Result. execute installs the handler, writes the tagged command, and bridges the handler's completion into the async world. This keeps the public API flat while all the streaming/parsing lives in per-command handlers.

Connection lifecycle

  • connect() bootstraps via ClientBootstrap.bootstrap(server:logger:group:), then immediately calls refreshCapabilities().
  • login(username:password:) runs LoginCommand and merges any post-login capabilities the server advertises.
  • Fetch comes in coarse (SequenceSet/UIDSet → MessageSet) and single-message (.complete) variants; UID-based fetch is the stable identifier.
  • Push: idle() returns an AsyncStream<IdleEvent>; done() ends it; noop() polls when not idling.

Errors are normalized through IMAPError(error) so NIO/SSL failures surface as IMAPError cases (notConnected, alreadyConnected, timedOut, capabilityNotSupported, …).

SMTP is a strict request/response conversation, so it's modeled as an explicit state machine inside a NIO ChannelInboundHandler. The handler tracks what it is waiting for with an Expect enum and advances on each server .ok:

private enum Expect {
    case initialMessageFromServer
    case okForHello            // EHLO
    case okForStartTLS
    case tlsHandlerToBeAdded
    case okForAuthBegin        // AUTH LOGIN
    case okAfterUsername
    case okAfterPassword
    case okAfterMailFrom
    case okAfterRecipient      // one per RCPT TO
    case okAfterData
    case okAfterTransferData
    case okAfterQuit
    case error(Error)
    case nothing
}

In channelRead, a .ok response is matched against the current Expect and the next command is written; anything else flips the machine to .error. The TLS branch is notable: on okForStartTLS it dynamically inserts an NIOSSLClientHandler into the pipeline (tlsHandlerToBeAdded) before continuing to AUTH — that's how STARTTLS upgrades an open plaintext connection.

Supporting handlers split responsibilities: RequestEncoder (commands→bytes), MessageHandler (line framing), ResponseHandler (parse codes), LoggingHandler (debug). SMTPClient.send(email:) sends to all recipients, looping RCPT TO per address.

Unlike IMAP, JMAP is stateless HTTP. A session is fetched once, then every operation is one or more methods POSTed as a batch.

// 1. Discover the session document
let session = try await JMAPClient.session(server: server)   // GET /.well-known/jmap

// 2. Call methods against the session's apiURL
let mailboxes = try await client.mailboxes()                 // Mailbox/get
let ids       = try await client.emails(in: inbox)           // Email/query
let emails    = try await client.emails(ids)                 // Email/get
let thread    = try await client.thread(for: email)          // Thread/get

Each call conforms to a Method protocol and encodes to the JMAP invocation tuple ["Email/get", { … }, "callId"]. A URLSession extension (jmapAPI/jmapSession) handles posting the batch and decoding the parallel response array into MethodGetResponse/MethodQueryResponse values. Filtering uses Filter / Email.Condition (inMailbox, hasKeyword, before/after, full-text). Authorization is a bearer token or OAuth2.

💡Why JMAP is faster Listing the inbox is one HTTP round-trip carrying Email/query + Email/get together, versus IMAP's connect → login → select → fetch sequence over a stateful socket.

A message is a Part: headers plus a body. When the ContentType is multipart, the Body splits on the boundary and recursively decodes each child into its own Part — so multipart/alternative inside multipart/mixed with attachments parses into a tree.

  • Part is RawRepresentable: Part(rawValue: Data) parses; part.rawValue re-serializes — symmetric parse/build.
  • ContentTransferEncoding decodes base64 / quoted-printable bodies to bytes.
  • ContentDisposition distinguishes inline content from named attachments.
  • Header lookups are fuzzy/case-insensitive to tolerate real-world senders.

This is the bridge between the wire (IMAP/SMTP bytes) and typed content the UI can render — the body that ReadEmailView hands to its WKWebView originates here.

The Account.Server struct is protocol-agnostic and — crucially — its credentials never live in the struct's Codable form. The authorization property is a computed bridge to the keychain:

public struct Server: Codable, Equatable, Hashable, Identifiable, Sendable {
    public var serverProtocol: ServerProtocol          // imap · jmap · smtp
    public var connectionSecurity: ConnectionSecurity  // == IMAP.ConnectionSecurity
    public var authenticationType: AuthenticationType
    public var username: String
    public var hostname: String
    public var port: Int

    public var authorization: Authorization {
        set { URLCredentialStorage.shared.set(authorization:
                  Authorization(user: user, password: newValue.password),
                  persistence: .permanent) }
        get { … URLCredentialStorage.shared.authorization(for: user) … }
    }

    // keychain key = "user@example.com IMAP:E621E1F8"  (unique per account+protocol)
    var user: String { … username + protocol + UUID suffix … }
}

So persisting an account writes only non-secret metadata to disk; the password/token round-trips through URLCredentialStorage keyed by a synthesized user that appends the protocol and a UUID suffix — letting the same address have distinct IMAP/SMTP/JMAP credentials.

Protocol bridging

Typed initializers convert the unified server into each protocol's own Server, enforcing capability rules along the way:

extension IMAP.Server { public init(_ server: Account.Server) throws { … } }
extension JMAP.Server { public init(_ server: Account.Server) throws {
    self.init(authorization: .bearer(server.authorization.rawValue),
              host: server.hostname, port: server.port) } }
extension SMTP.Server { public init(_ server: Account.Server) throws { … } }

// guards surface as protocol-specific errors:
IMAPError.oAuth2NotSupported / .serverProtocolMismatch
JMAPError.oAuth2NotSupported / .serverProtocolMismatch / .sessionNotFound

For example, IMAP with an OAuth2-only auth type throws IMAPError.oAuth2NotSupported — the mismatch is caught at bridge time, not at connect time. The Account caches the resulting IMAPClient/JMAPClient by UUID and reuses it.
